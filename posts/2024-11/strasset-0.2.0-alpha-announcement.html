STRAsset v0.2.0 alpha 的更新預告

前文

本來作為此插件的名稱 TRScript 因為不好與物件腳本 TRScript 進行分辨，所以目前暫時更改為在虛幻引擎中使用的名稱 STRAsset。

本來將插件取作 TRScript 只是因為目前正在進行的遊戲專案的縮寫是 TR ，然後加上腳本的英文 Script 就形成了這個名稱。

原本在虛幻引擎中的名稱 STRAsset 則是目前工作室暫定的名稱的縮寫 STR 加上 Asset 而組合成的，之後也會將前方的 STR 修改為正式的工作室的名稱的縮寫。

這裡也要說聲抱歉，因為這樣修改名稱可能會對於一些工作的進度造成延後，還請接受。


v0.2.0 alpha - 物件與特效 -

這次的 v0.2.0 alpha 的主要要素為 物件與特效 ，主要的內容是將角色腳本的功能進行衍生，從角色腳本變更為了物件腳本，而角色腳本只是其中的分支。

會進行這項更動的起因還蠻複雜的，總結成幾句話大概如下：

有天我正在思考關於飛行道具的狀況時，我查詢了 Arc System Works 在製作飛行道具的相關資料。

在研究了一番過後，我發現了 Arc System Works 用於製作角色、飛行道具與特效是使用自行研發的BBS腳本來控制的，而且內部的命令也都有很大的重疊率。

所以我花了一些時間將部分功能從 STRChara (角色物件) 中獨立了出來，並且製作了一個新的父物件 STRObject。

STRObject 包含了物件腳本可以通用的參數、函式與命令。

如果在未來使用了 TRScript 創作了其他類型的腳本，就可以透過繼承 STRObject 來縮短工作流程與時間。


v0.2.0 alpha 中的細微修改優化

在 v0.2.0 中，對於了一些不太順手的部分進行了優化

以下為在這次的更新中進行的優化：

  • 優化了 elseif ，將其簡短為 elif
  • 優化了輸入緩衝，目前還不確定會不會出現記憶體遺失等錯誤
  • 優化了狀態的執行邏輯
  • 優化了角色的二段跳，將會在第二段改變面向

v0.2.0 alpha 中會處理的修正

在進行 v0.2.0 的製作時，我也發現了一些之前沒有發現的錯誤與問題，並且進行了修正。

以下為在這次的更新中進行的修正：

  • 修正了角色在面相左側時(2P側)的2段跳方向錯誤
  • 修正了角色的行為結束時的狀態會變更為 "" 的問題

Github 頁面於未來的更動

既然創建了一個類似官方文檔的頁面，就開始顯得 Github 的單頁簡介缺少了很多實用的功能，而且一次更新兩處也會增加我的負擔。

所以 Github 的頁面也會在更新發布的之後簡化與停止更新，只會留下簡介跟文檔的超連結。


結語

到了這裡，這篇公告也差不多進入了尾聲。

雖然之前將 STRAsset 發布了，但也是在發布的不久後發現了其中有很多沒有發現與修正的錯誤，所以在這幾天我都花了些時間進行了部分的修正。

也將基礎的架構修改成了之後的 v0.2.0 alpha 的樣貌，未來在製作新的腳本物件應該會變得更輕鬆了。

在 v0.2.0 alpha 的內容穩定之後，將會在幾天至幾周內將新的版本公布。 文檔目前也在著手更新 v0.2.0 alpha 的內容，並且會盡可能地跟 v0.2.0 alpha 在同時間完成。


這邊也要再一次的感謝回到了這個網站的你，或是第一次來到這個網站的你，以後還請多多指教了。











不是很重要的閒聊

現在在閒暇時間都會思考一個更有辨識性的暱稱，畢竟目前正在使用的是以前隨便亂用而產出來的。

我應該是不會用個甚麼筆名之類的，應該會用英文的暱稱，目前的目標是比較正式跟有辦法閱讀，而不是奇怪的符號。

我也會花一些時間在工作室的正式名稱跟專案的正式名稱上，畢竟現在只是用一個暫時的名稱撐過還沒有想法的時間。雖然有幾個點子，但是這些都沒有達到我的標準。

目前我心中的名列前茅的選項是由High Skill Ceiling 縮寫而來的 HSC 或是 HC，也是我在製作遊戲時的最終目標。

High Skill Ceiling 是技術上限高的英文，有著這個類型的遊戲通常都需要花費大量的時間來精通，而我喜歡的遊戲也都有著這個特徵。

雖然也有想過朝著技術下限低發展，但是這樣會讓遊戲的技術在還沒到達中半段的時候就已經被熟悉了，這剛好跟我喜愛的類型是完全相反的。

因此，我從最剛開始學習遊戲開發時，我就朝著系統複雜或是技術需求高的目標前進，為了讓系統有趣而絞盡腦汁也是當初設計遊戲的樂趣之一。

我一路從以前想要開發複雜的3D射擊遊戲一路走到現在想要開發精緻又有深度的2D格鬥遊戲，我也算是開始真正的往一直以來的目標前進了吧。

不知道看到了這裡的你，有沒有往一直以來的目標前進呢?

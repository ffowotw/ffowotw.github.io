創建角色

本篇教學將會教導您該如何創建出一個可遊玩的角色。

重要

下方會以XXX代表角色的名稱／縮寫。

請將 XXX 替換為您的腳色的名稱／縮寫。

1. 在專案中建立角色的資料夾

一個整齊的工作環境，不外乎有一個整齊的資料夾架構。 一個好的資料夾架構不只可以讓整體變得更整齊，也方便之後找到想要修改的檔案

好的資料夾架構

一個好的資料夾架構讓人可以清楚的知道每個檔案應該的位置

└─ Content
   └─ Chara
      ├─ ...
      ├─ XXX
      │  ├─ Animation
      │  ├─ Data
      │  ├─ Material
      │  └─ Mesh
      ├─ YYY
      │  ├─ Animation
      │  ├─ Data
      │  ├─ Material
      │  └─ Mesh
      └─ ...

差的資料夾架構

一個差的資料夾架構會讓整體變的混亂

└─ Content
   └─ Chara
      ├─ ...
      ├─ AAA
      │  ├─ Data
      │  ├─ Material
      │  └─ Mesh
      │     └─ Animation
      ├─ BBB
      │  ├─ Animation
      │  └─ Mesh
      │     ├─ Data
      │     └─ Material
      └─ ...

2. 建立並匯入角色腳本

此插件的角色腳本獨立於虛幻引擎4之外，因此需要再專案外部創建插件並匯入。

請找一個資料夾或創建一個新的資料夾，並在其中新增名為 SCR_XXX.trscript 的角色腳本。

接著，請參考或是複製以下的範本進入該腳本中：

beginFunction: "PreInit"
    charaName: "XXX"
endFunction:

	life: 10000
	weight: 100
	defence: 0
	airJumpCount: 1
	airDashCount: 1
	walkFSpeed: 6125
	walkBSpeed: 4812
	dashFInitSpeed: 15000
	dashFAcceleration: 700
	dashFriction: 95
	dashBXSpeed: 15750
	dashBYSpeed: 5250
	dashBGravity: 700
	jumpFXSpeed: 8750
	jumpBXSpeed: 8750
	jumpYSpeed: 35000
	jumpGravity: 1925
	highJumpFXSpeed: 8750
	highJumpBXSpeed: 8750
	highJumpYSpeed: 45500
	highJumpGravity: 1925
	airDashMinHeight: 105000
	airDashFTime: 20
	airDashBTime: 10
	airDashFSpeed: 30625
	airDashBSpeed: 24500
	airDashFNoAttackTime: 5
	airDashBNoAttackTime: 5
	pushboxWidthStand: 105000
	pushboxHeightStand: 227500
	pushboxWidthCrouch: 112000
	pushboxHeightCrouch: 175000
	pushboxWidthAir: 105000
	pushboxHeightAir: 192500
	pushboxHeightLowAir: -70000

修改好角色腳本之後，請將該腳本匯入至Data的目錄下。這是如果出現了 SCR_XXX 就代表成功了。

如果匯入失敗，請檢查插件編輯介面中是否存在與啟用此插件 STRAsset 。

3. 建立剩餘的要素

在成功的將角色腳本匯入專案之後，請將角色的模型、動畫、材質分別匯入至：Mesh、Animation 與 Material 中。

完成後請在Mesh的目錄下新增一個 STR Asset/MeshArray ，接著將匯入的模型新增至剛剛創建好的 MeshArray 裡面，並且替各個部分取好名稱，之後會在角色腳本使用到這些名稱。

然後在Animation的目錄下新增 STR Asset/AnimSet 與 STR Asset/AnimArray ，先將各個部件的動畫新增至對應的 AnimSet 中，再將 AnimSet 加入 AnimArray ，並且取好對應的部件的名稱。

最後，在Animation的目錄下新增一個繼承了 STRAnimInstance 的動畫藍圖。

因為此插件的動畫系統會將動作固定在特定的時間，所以需要將動畫轉換為一幀動畫。
